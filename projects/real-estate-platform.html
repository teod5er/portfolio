1. Shared survey view

Problem

The existing view relied on an admin interface with editing options disabled.

Goal

The goal was to create an interface that mimics the experience of a booklet - quick overview with all properties and a explore page with interactive map features ans.

The same screen is used for managing real estate broker tasks and showcasing properties to clients. There was no customization or personalization.
The experience of browsing the brochure is created with two screens — Summary view shows all the properties, while Explore view allows for a more detailed view with interaction on the map.
The fixed position of the Tab switch enables users to switch modes at any time.
The navigation bar keeps its features but presents them more user-friendly.
Each screen is fully responsive and optimized for mobile view.

2. Survey analytics

Problem

Brokers often rely on direct feedback from clients about proposed properties, such as comments or calls. That back-and-forth approach slows down their work significantly.

Goal

The goal was to create an interface that solves this problem and gives brokers a simple overview to help them during their next calls with clients.

Analytics builds on the report that the broker sends to the client, providing insight into the client's overall interactions.
The broker understands the client's current status in this deal.
The broker gets details about the property clients engage with the most.

3. Team Insights

Problem

The current platform doesn't let admins view all team and broker statistics in one place. They have to check each profile manually for the needed metrics.

Goal

Allow admins to see the workload, number of reports created, closed jobs, and financial metrics for a specific period.

The page is designed to give users insight into selecting a time period and broker.
The admin can see the list of all clients on the platform.
The admin can monitor the broker's activities.
