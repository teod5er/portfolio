2. UX Problems

Problem

1. Upgrade existing flow
The solution has to be an upgrade to the existing UI design system, but keep existing flows.

2. Promote
The new flow must clearly promote the split payment option.

3. Full flexibility
The user must be able to combine multiple payments with special plan that has separate flows inside payment flow.

4. Accessible
The system must be easy to use and should handle possible fallback scenarios. User should feel safe.

4. User flow

User flow

The existing user flow is updated with the new split payment option. This way, the project didn't require a complete overhaul of the complex payment system.

User flow integrated split payment option within current payment flow.
